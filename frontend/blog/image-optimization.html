Modern Formats: WebP & AVIF

WebP

  • 25-35% smaller than JPEG
  • Supports lossy and lossless compression
  • Supports animation and transparency
  • Excellent browser support
<picture>
<source srcset="image.webp" type="image/webp">
<img src="image.jpg" alt="...">
</picture>

AVIF

  • 50% smaller than JPEG
  • Better quality than WebP
  • HDR support
  • Growing browser support
<picture>
<source srcset="image.avif" type="image/avif">
<source srcset="image.webp" type="image/webp">

Compression Techniques

Lossy vs Lossless Compression

Lossy Compression

Removes some data to reduce file size. Best for photos where slight quality loss is acceptable.

Use: Quality 75-85 for JPEG

Lossless Compression

Reduces size without quality loss. Best for graphics, logos, and images with text.

Use: PNG-8 or WebP lossless

Progressive JPEGs

Progressive JPEGs load in layers, showing a blurry version first that sharpens progressively:

  • Perceived faster loading
  • Better user experience on slow connections
  • Slightly larger file size (negligible with good optimization)

Responsive Images

Serve different image sizes based on the user's device:

<img
src="image-800.jpg"
srcset="
image-400.jpg 400w,
image-800.jpg 800w,
image-1200.jpg 1200w,
image-1600.jpg 1600w"
sizes="(max-width: 600px) 100vw, 50vw"
alt="Responsive image"
>

Tip: Use image breakpoints that match your CSS breakpoints for optimal performance.

Lazy Loading Images

Load images only when they enter the viewport:

<!-- Native lazy loading (recommended) -->
<img src="image.jpg" loading="lazy" alt="...">

<!-- Eager loading for above-fold images -->
<img src="hero.jpg" loading="eager" alt="...">

Benefits

  • Faster initial page load
  • Reduced bandwidth usage
  • Better Core Web Vitals scores
  • Native browser support (no JS needed)

Caveats

  • Don't lazy load LCP images
  • Reserve space with aspect-ratio
  • Fallback needed for older browsers
  • Can impact ad revenue
